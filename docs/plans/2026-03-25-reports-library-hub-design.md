# Reports Library Hub — Design Document

**Date:** 2026-03-25
**Status:** Approved
**Scope:** Replace the "coming soon" on `/auditor/reports` with a cross-audit reports library.

---

## Problem

The auditor's top-level Reports page (`/auditor/reports`) shows "Reports coming soon." Auditors currently have to drill into each individual audit (`/auditor/audits/[auditId]/report`) to find, view, or manage reports. There is no single place to see all reports across all companies.

---

## Goal

Turn `/auditor/reports` into a **Reports Library Hub**: a searchable, filterable list of every audit report across all companies, where each row links through to the existing per-audit report editor.

---

## Architecture

### New API endpoint

`GET /api/audit-reports` (top-level, no `[id]`) — returns all `AuditReport` rows joined with their parent `Audit` and `Company`. Supports optional query params:

- `?status=DRAFT|FINAL` — filter by report status
- `?companyId=<id>` — filter by company
- `?q=<string>` — search by company name

**Existing endpoints are untouched:** `GET /api/audit-reports/[id]`, `PATCH`, `POST /generate`, etc. all remain as-is.

### Page

`/auditor/reports/page.tsx` — converted from the current stub to a proper page that:
1. Fetches from `GET /api/audit-reports`
2. Renders filter bar + table
3. Each row navigates to `/auditor/audits/[auditId]/report`

No DB schema changes required.

---

## UI

### Filter bar
- **Text search** input — filters by company name (client-side)
- **Status** dropdown — All / Draft / Final
- **Company** dropdown — populated from distinct companies in the result set

### Table columns
| Column | Notes |
|---|---|
| Company | Company name |
| Audit Year | From the parent Audit record |
| Status | Badge: orange `DRAFT`, green `FINAL` |
| Version | `v1`, `v2`, etc. |
| Generated By | Claude / Gemini |
| Last Modified | Relative date (e.g. "2 days ago") |
| → | Link icon to open in editor |

### Empty state
If no reports exist, show a helpful message: *"No reports yet. Open an audit and use Generate with AI to create the first one."* with a link to `/auditor/audits`.

### Styling
Uses the app's zinc/orange design tokens (matching the rest of the auditor portal). Not the old gray styles currently on the per-audit report page.

---

## Out of Scope

- **Report generation from the hub** — generation stays in the per-audit editor (`/auditor/audits/[auditId]/report`). The hub is browse + navigate only.
- **AI writer customization** (colour scheme, style preset pickers) — belongs to the existing `2026-03-25-ai-report-writer-customization.md` plan; implemented in the per-audit editor.
- **Pagination** — not needed at current data volumes.

---

## Success Criteria

1. `/auditor/reports` loads without errors and shows all existing audit reports.
2. Filtering by status and company narrows the list correctly.
3. Text search filters by company name.
4. Clicking any row navigates to the correct per-audit report editor.
5. Empty state is shown when no reports exist.
6. Page matches the app's zinc/orange design system.
